Badge Priority & Hierarchy

When multiple badges could apply, show only the highest priority. Never stack multiple badges on one button.

Priority Badge Type Reason Example
1. Highest ! Exclamation Time-sensitive, potential loss Event ends in 3 hours
2. High ↑ Green Arrow Immediate progression available Hero can level up now
3. Medium # Number Badge Multiple actions waiting 5 quests ready to claim
4. Medium "NEW" Badge Fresh content to explore New event unlocked
5. Lowest ● Red Dot General notification Daily reward available

✅ Best Practices (Anti-Fatigue Design)

  • Auto-Clear When Action Completed: Badge disappears instantly when player claims/upgrades (Clash Royale pattern)
  • NEW Badge Timeout: Automatically remove "NEW" after 24-48 hours or first visit (Marvel Strike Force)
  • One Badge Per Button: Never stack multiple indicators - show highest priority only
  • Meaningful Numbers Only: Show count (12) only when >1. Use red dot for single items
  • Contextual Urgency: Only use ! exclamation for genuine time pressure (<24hrs to expiry)
  • Tutorial Exception: Suppress all badges during tutorial/onboarding to avoid overwhelm
  • Smart Grouping: Combine similar notifications (3 heroes ready = 1 badge on Heroes screen)
  • User Control: Allow dismissal of "NEW" badges manually (long-press to clear)
  • Session Limit: Cap total active badges at 5-7 across entire UI to prevent Christmas tree effect
  • No False Urgency: Don't abuse ! exclamation for non-expiring shop offers (Whiteout Survival does this well)

❌ Anti-Patterns (Causes Fatigue)

  • Permanent Badges: Never show badges that never clear (creates learned helplessness)
  • Over-Notification: Badge on every screen/button even for minor actions
  • Fake Urgency: ! exclamation on permanent shop items or non-expiring content
  • Badge Spam: Same notification shown in 3+ places simultaneously
  • No Hierarchy: All notifications look equally important (everything urgent = nothing urgent)
  • Hidden Dependencies: Badge shows but action impossible due to hidden requirement
  • Inconsistent Logic: Badge appears/disappears randomly without clear trigger
  • Tutorial Overload: Showing 10+ badges to new player immediately
  • Clickbait Badges: Badge leads to empty/already-cleared screen
  • No Feedback: Badge persists even after player completes the action

Contextual Badge Application

🏠 Main Hub Screen

  • Heroes Button: ↑ Green arrow if 1+ heroes can upgrade with current resources
  • Quests Button: # Number showing claimable rewards (e.g., "3")
  • Shop Button: "NEW" for first 24h after shop refresh, then ● red dot if items available
  • Events Button: ! Exclamation if main event ends in <24h, else "NEW" for first 48h
  • Mail Button: # Number for unread count, max display "99+"
  • Guild Button: ● Red dot for new messages/requests (no number to avoid spam)

⚔️ Heroes Screen

  • Individual Hero Portrait: ↑ Green arrow only if that specific hero can level/rank up NOW
  • Filter Tabs (All/Tank/DPS): # Number showing upgradeable heroes in that category
  • New Hero (first 24h): "NEW" badge, then clear automatically
  • Awakening Tab: ! Exclamation if awakening materials expire soon, else ↑ green if ready

🎁 Daily Rewards

  • Login Calendar: ● Red dot when claimable, disappears after claim
  • Free Chest: ● Red dot when timer complete (not before)
  • Video Rewards: # Number showing available watch count (e.g., "3/5")
  • Pass/Battlepass: # Number for all unclaimed tier rewards combined

💰 Shop/Store

  • Daily Deals: "NEW" for first 2 hours after refresh, then nothing
  • Flash Sale: ! Exclamation only if <4 hours remaining
  • Event Shop: "NEW" when event launches, ● red dot if currency available to spend
  • Regular Shop: No badge (opt-in browsing, not push notification)

⚡ Energy/Stamina System

  • Campaign Button: No badge normally (even at full energy - Clash Royale pattern)
  • Campaign Button (capped): ! Exclamation only if energy 100% full for >30min
  • Campaign Button (new content): "NEW" badge for new chapter unlock, 48h timeout

Badge Timing & Lifecycle

Appearance Triggers

  • Instant: Reward becomes claimable, quest completes, timer finishes
  • Session Start: Check for new mail, event updates, daily reset items
  • Real-Time: Guild messages, PvP attack notifications (push notifications)
  • After Action: Hero levels up → show upgrade badge on Heroes screen

Disappearance Rules

  • Immediate: Player completes the action (claim/upgrade/view)
  • 24-Hour Auto-Clear: "NEW" badges on repeating content (daily shop)
  • 48-Hour Auto-Clear: "NEW" badges on permanent unlocks (new feature/mode)
  • Manual Dismiss: Allow long-press to clear "NEW" if player wants
  • Conditional Clear: Green ↑ disappears when resources spent elsewhere

Update Frequency

  • Real-Time: Red dots, upgrade arrows (check after every action)
  • Every 5 Minutes: Energy caps, timer completions (background check)
  • Session Check: "NEW" badges, mail counts (on app open)
  • Daily Reset: Shop refresh, daily quest badges (4am UTC typical)

Platform-Specific Considerations

iOS vs Android

  • iOS App Icon Badge: Mirror most urgent in-game badge number (mail/quests)
  • Android Home Widget: Show daily reset timer + most important action
  • Push Notifications: Only for time-sensitive (energy full, event ending, PvP attack)
  • Badge Numbering: iOS convention = show count, Android = red dot more common

Accessibility

  • Color-Blind Mode: Add symbols to badges (! ↑ # NEW) not just colors
  • Screen Readers: Badge announcements like "Upgrade available" not just "badge"
  • Reduced Motion: Static badges, no pulse/bounce animations
  • High Contrast: Border around badges for visibility

Summary: 5 Core Principles

🎯 Golden Rules for Badge UX

  1. Meaningful Only: Every badge must represent a real, completable action
  2. Clear Hierarchy: One badge type per button, prioritized by urgency
  3. Auto-Clear: Badges disappear when action is taken or timeout expires
  4. Respect Attention: Cap total badges at 5-7, no spam, no fake urgency
  5. Instant Feedback: Badge state changes immediately after player action
