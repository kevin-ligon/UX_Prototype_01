Pro Tips for Best Results
✅ Show Multiple States
Idle, pressed, disabled - helps recreate exact behavior
✅ Explain the Feeling
"Snappy" vs "Weighty" - describes interaction intent
✅ Use References
"Like Clash Royale's menu" - speeds up understanding
✅ Focus on Key Interactions
Start with 1-2 critical flows, expand later
❌ Don't Send Everything
50 screens at once = overwhelm. Iterate step by step
❌ Don't Assume Context
Explain game-specific terms and interactions clearly
✨ Perfect Request Example
"I'm designing a mobile RPG inventory system.

Assets: [uploads 3 wireframe images]
• 01_inventory_closed.png
• 02_inventory_open.png
• 03_item_selected.png

UI Art: In /assets folder
• panel_bg.png
• button_close.png
• item_icons/ (10 icons)

Interaction to test:
1. Tap inventory button → panel slides up from bottom
2. Tap item → detail panel expands from item
3. Tap outside → everything dismisses

Focus: Slide-up animation needs to feel smooth on mobile
Can you add timing controls so I can tune it?

Style: Fantasy theme, gold/purple colors
Mobile-first, portrait orientation"
Result: Working prototype delivered in ~5 minutes, ready to test on any device
